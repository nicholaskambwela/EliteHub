Section 01

What is an Operating System?

Definition An Operating System (OS) is system software that acts as an intermediary between the user and computer hardware. It manages hardware resources and provides a platform for application software to run.

Think of it this way: your hardware is the kitchen, your apps are the chefs, and the OS is the head chef that assigns stations, manages equipment, and makes sure nobody's fighting over the stove.

🧑‍💼
Resource Manager
Allocates CPU time, memory, and storage to programs so they don't step on each other.
🔌
Hardware Abstraction
Apps don't talk to hardware directly — the OS handles that. Your browser doesn't need to know what brand of RAM you have.
🛡️
Security Gatekeeper
Controls which users and programs can access what. User accounts, permissions, passwords — all OS territory.
🖥️
User Interface
Provides either a graphical (GUI) or text-based (CLI) interface for users to interact with the computer.

Section 02

The 5 Main Functions of an OS

⚠️ Exam Alert — 2025 Q4a, 2015 Q1b, 2024 Q5 "Explain the main functions of an operating system" — this has appeared in EVERY exam. Know all 5 cold.

The OS decides which program runs at what time. Since the CPU can only execute one instruction at a time (mostly), the OS uses scheduling to give each program a slice of CPU time.


Key terms: Process, multitasking, scheduling, context switching.


Exam answer tip: "The OS manages processes by scheduling CPU time, creating and terminating processes, and handling deadlocks."

The OS allocates and deallocates RAM to programs. It keeps track of which parts of memory are in use and which are free.


When RAM runs out, the OS uses virtual memory — it temporarily moves less-used data to the hard drive to free up RAM. That's why your computer slows down when you have 50 tabs open.


Exam answer tip: "The OS allocates RAM to processes, frees memory when processes terminate, and uses virtual memory when RAM is insufficient."

The OS organises data into files and directories (folders). It handles creating, reading, updating, and deleting files. It also manages file permissions — who can read, write, or execute each file.


File attributes: Read-only, Hidden, System, Archive.


Exam answer tip: "The OS manages the file system by organising files into directories, controlling file access through permissions, and maintaining file metadata."

The OS communicates with hardware devices using device drivers. A driver is a small program that tells the OS how to talk to a specific device (printer, mouse, graphics card etc.).


Without drivers, your OS wouldn't know what to do with your hardware. That's why you sometimes need to install drivers when you plug in new hardware.


Exam answer tip: "The OS manages I/O devices through device drivers, coordinates data transfer between devices and the CPU, and handles interrupts."

The OS enforces access control — making sure only authorised users can access certain files or run certain programs. It manages user accounts, passwords, and permission levels.


Also includes: protecting against malware, logging system events, and managing network access.


Exam answer tip: "The OS provides security through user authentication, access control lists, and protection against unauthorised access to system resources."

Section 03

Common Operating Systems

🪟
Windows
Developed by Microsoft. Most widely used desktop OS. GUI-based. Used in homes, schools, businesses. Proprietary (you pay for it).
🐧
Linux
Open-source. Highly stable. Used heavily in servers. Command-line focused but has GUI versions (Ubuntu, Fedora). Free to use.
🍎
macOS
Developed by Apple. Unix-based. Only runs on Apple hardware. Known for clean GUI. Used widely in creative industries.
🤖
Android
Linux-based mobile OS by Google. Open-source. Powers most smartphones worldwide. Touch-based interface.
📱
iOS
Apple's mobile OS. Only runs on iPhones/iPads. Closed ecosystem. Known for security and smooth performance.
⚙️
Unix
The grandfather of modern OS. Highly stable. Foundation for both Linux and macOS. Primarily used in servers and mainframes.
Windows vs Linux — Exam Comparison Windows: commercial, GUI-first, widely used, proprietary. Linux: open-source, CLI-first, server-focused, free, highly stable. macOS: Unix-based, Apple-exclusive, GUI, used in creative fields.

Section 06

File Management & File System

⚠️ Exam Alert — 2025 Q4b, Q4c "Describe the structure of a file system including directories and paths" + "Explain the significance of file attributes."
📁
File
A named collection of related data stored on a storage device. Has a name, extension, and metadata (size, date, permissions).
🗂️
Directory (Folder)
A container that organises files and other directories. Forms the tree structure of the file system.
🗺️
Path
The location of a file/folder. Absolute path starts from root (C:\Users\Nicholas\). Relative path starts from current location.
🔑
File Extension
Tells the OS what type of file it is. .docx = Word document, .mp3 = audio, .html = webpage, .exe = executable program.

File Attributes

AttributeMeaningExample Use
Read-only (R)File can be read but not modifiedSystem configuration files
Hidden (H)File is not visible in normal directory listingOS system files
System (S)File is part of the operating systemboot files, kernel files
Archive (A)File has been modified since last backupBackup software uses this

File Operations

📋
Copy vs Move
Copy — creates a duplicate; original stays. Move — transfers to new location; original is removed.
🗑️
Delete & Recovery
Deleted files go to Recycle Bin (Windows) / Trash (Mac). Permanently deleted only when bin is emptied. Recovery is then very difficult.
🔐
File Permissions
Read, Write, Execute, Delete — the OS controls who can do what to a file based on user accounts and groups.

Past Paper Questions & Model Answers

2025 Final Explain the main functions of an operating system. 8 marks

An operating system performs five core functions:

Model Answer (8 marks)
  • Process Management — The OS schedules and manages CPU time for all running programs, handling multitasking through time-slicing.
  • Memory Management — Allocates and deallocates RAM to processes; uses virtual memory when RAM is insufficient.
  • File System Management — Organises data into files and directories; manages file creation, access, modification, and deletion.
  • Device Management — Communicates with hardware via device drivers; controls I/O operations and interrupts.
  • Security & User Management — Controls user access through authentication and permissions; protects system resources from unauthorised access.
2025 Final Describe the structure of a file system, including directories and paths. 8 marks
Model Answer
  • A file system is the method an OS uses to organise and store data on a storage device.
  • It uses a hierarchical tree structure: root directory → subdirectories → files.
  • A directory (folder) is a container holding files and other directories.
  • An absolute path specifies the full location from root, e.g. C:\Users\Nicholas\Documents\notes.txt
  • A relative path specifies location relative to the current directory, e.g. Documents\notes.txt
  • Files have metadata: name, extension, size, date created/modified, permissions.
  • Common file systems: NTFS (Windows), ext4 (Linux), FAT32 (USB drives).
2025 Final Explain the significance of file attributes: Read-only, Hidden, System, Archive. Part of Q4c
Model Answer
  • Read-only — Prevents accidental modification; used for files that should not be changed, like configuration files.
  • Hidden — File is not displayed in normal directory view; protects sensitive system files from accidental deletion.
  • System — Marks files critical to OS operation; prevents deletion or modification by regular users.
  • Archive — Indicates file has been modified since last backup; used by backup software to identify files that need backing up.
2015 Final State 5 services provided by the Operating System, giving a brief explanation of how the OS provides each service. 10 marks
Model Answer (2 marks each)
  • Program execution — OS loads programs into memory and runs them, managing their lifecycle from start to finish.
  • I/O operations — OS provides a standard way for programs to use I/O devices without needing device-specific code.
  • File system manipulation — OS provides system calls to create, read, write, delete, and navigate files and directories.
  • Communication — OS enables processes to communicate via shared memory or message passing.
  • Error detection — OS constantly monitors for errors in CPU, memory, I/O devices, and user programs, taking corrective action.
2025 Q1c–e Define: Operating System, Application Software. State 3 OS functions. Explain file extensions. 10 marks
Model Answer
  • Operating System — System software that manages computer hardware and software resources, providing a platform for application programs to run.
  • Application Software — Programs designed to help the user perform specific tasks, such as word processing, browsing, or playing games. Examples: MS Word, Chrome, VLC.
  • 3 OS Functions — Process management, memory management, file system management (or any 3 from the 5).
  • File Extension — A suffix added after a filename (after the dot) that identifies the file type and tells the OS which program should open it. Example: .docx = Word, .pdf = PDF reader, .mp3 = audio player.
